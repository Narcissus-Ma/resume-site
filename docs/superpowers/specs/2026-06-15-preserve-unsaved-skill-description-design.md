# 保留未保存技能描述设计

## 问题

技能描述输入框由 Ant Design Form 管理，新增按钮却基于 React 状态中的旧 `data` 追加内容。调用 `updateData` 后，旧数据会重新写回 Form，导致用户尚未保存的输入被覆盖。

## 方案

新增技能描述时，先通过 `form.getFieldsValue(true)` 获取表单实时值，并以它作为当前简历数据，再追加默认描述“新的技能描述”。合并与追加逻辑放入纯规则函数，Hook 只负责读取 Form 和更新状态。

## 测试

回归测试构造旧数据和包含未保存输入的表单数据，验证追加后保留表单输入，并且不修改原始对象。
