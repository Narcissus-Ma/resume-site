# 简历网站主题与响应式优化设计

## 背景

当前网站由首页与简历详情页组成，使用 React、TypeScript、Tailwind CSS 和 Ant Design。现有夜间模式主要通过组件内条件类名实现，但 Ant Design 的默认主题仍使用浅色模式文字，导致导航、标题、进度文字等元素在深色背景上显示为黑色。移动端顶部导航无法收缩，在 390px 视口下产生横向溢出。

本次改造保留现有页面结构、内容数据、主要动画与组件职责，集中解决主题一致性、文字可读性和移动端适配问题。

## 目标

- 首页与简历详情页使用统一的亮色、暗色主题。
- 暗色主题采用深蓝灰背景、高对比文字和蓝色强调色。
- 默认跟随系统主题，支持手动切换并记住用户选择。
- Ant Design 与 Tailwind 样式使用同一套语义颜色。
- 普通正文对比度达到 WCAG AA 的 4.5:1，大字号文字达到 3:1。
- 390px 移动端不出现横向滚动，导航与首屏内容可正常使用。
- 打印与 PDF 始终使用清晰的浅色样式。

## 非目标

- 不重写网站信息架构。
- 不更换 React、Ant Design 或 Tailwind CSS。
- 不重做简历数据结构、编辑器和后端接口。
- 不引入完整的新 UI 模板或大型动画库。

## 主题架构

### 语义令牌

在全局样式中定义语义化 CSS 变量，并通过根元素的 `data-theme` 属性切换：

- `--color-page-bg`：页面主背景。
- `--color-section-bg`：交替区块背景。
- `--color-surface`：卡片、弹窗等表面背景。
- `--color-surface-muted`：标签、次级容器背景。
- `--color-text-primary`：标题和主要正文。
- `--color-text-secondary`：说明、时间、辅助信息。
- `--color-text-muted`：低优先级提示。
- `--color-border`：卡片和控件边界。
- `--color-primary`：主要操作与强调色。
- `--color-primary-hover`：强调色悬停状态。
- `--color-focus-ring`：键盘焦点环。

亮色主题使用浅灰页面、白色表面和深灰文字；暗色主题使用深蓝灰页面、略浅的卡片表面、偏白主文字和浅灰次文字。强调色在暗色模式下提高亮度，避免蓝色在深色背景中发闷。

### Ant Design

在应用顶层增加主题 Provider：

- 亮色模式使用 Ant Design 默认算法。
- 暗色模式使用 `darkAlgorithm`。
- 将主色、背景、文字、边框和圆角映射到语义令牌对应色值。
- 统一修复 Anchor、Typography、Card、Progress、Button、Select、Modal、Tooltip 等组件。

页面组件不再依赖大量 `darkMode ? ... : ...` 颜色拼接，而使用语义类或 CSS 变量。仅当布局或图片处理确实不同，才保留主题条件逻辑。

## 主题状态

主题状态支持三种内部来源：

1. 用户没有保存选择时，读取 `prefers-color-scheme`。
2. 用户点击主题按钮后，保存 `light` 或 `dark` 到 `localStorage`。
3. 用户没有保存选择时，持续监听系统主题变化并同步页面。

主题应用到 `document.documentElement.dataset.theme`，并同步 `color-scheme`，确保浏览器原生控件与页面主题一致。首页与简历详情页共享同一 Theme Provider 和切换逻辑。

## 页面设计

### 首页

- 顶部导航保持吸顶和毛玻璃效果，但使用主题表面色、边框和正确的文字颜色。
- 桌面端保留横向锚点导航。
- 移动端隐藏完整锚点列表，提供单一菜单按钮或紧凑导航面板，语言与主题按钮保留。
- 首屏保留左右布局与头像，将过大的垂直空白收敛到一个视口内。
- 姓名、职位、简介使用清晰的三级文字层级。
- 技能、经历、项目和联系区块继续交替使用背景，但差异通过页面背景与表面背景表达。
- 卡片减少暗色模式中的重阴影，改用细边框和轻微高光。
- 项目轮播与进度条的文字、按钮和指示器使用主题令牌。

### 简历详情页

- 使用与首页相同的主题 Provider 和主题切换入口。
- 页面背景、内容卡片、标签、时间和正文统一使用语义令牌。
- 顶部操作区在移动端允许换行，避免导出和打印按钮挤压标题。
- 保留当前项目、工作经历、教育和站点链接结构。
- 链接和技能标签在两种主题下保持足够对比度，并提供明确悬停、焦点状态。

### 打印

- `@media print` 下强制白色背景和深色文字。
- 隐藏导航、主题切换、编辑、导出、打印等交互控件。
- 移除阴影与不必要边框，防止暗色主题污染打印结果。
- 保持已有页边距和正文可读字号。

## 响应式规则

- `>= 768px`：使用桌面横向导航和首页左右首屏布局。
- `< 768px`：使用紧凑导航，首屏改为单列，主要操作按钮保持可点击尺寸。
- 页面根元素和主要区块设置合理的 `min-width: 0`，图片、轮播和长文本不得撑宽视口。
- 使用 Flex/Grid 完成布局，不使用 `float`。
- 以 390x844 为主要移动端验收视口，同时检查 768px 和默认桌面视口。

## 可访问性

- 普通文字对比度至少 4.5:1，大文字至少 3:1。
- 所有图标按钮提供可访问名称。
- 键盘焦点使用明显的统一焦点环。
- 主题按钮通过 `aria-label` 表达将要切换到的主题。
- 移动菜单具有正确的展开状态，支持键盘操作。
- 尊重 `prefers-reduced-motion`，减少非必要位移和循环动画。

## 实施边界

优先修改主题 Provider、主题 Hook、全局样式和现有页面组件。保持组件单一职责；若移动导航逻辑使 Header 过重，可拆分独立的移动菜单组件，文件名使用 kebab-case。

不修改简历内容、不增加新的网络请求、不调整接口协议。API 错误与认证逻辑保持原状。

## 验证

### 自动检查

- `pnpm typecheck`
- `pnpm lint`
- `pnpm format:check`
- `pnpm build`

### 浏览器检查

- 桌面端亮色与暗色首页。
- 桌面端亮色与暗色简历详情页。
- 390px 移动端首页无横向滚动。
- 390px 移动端简历详情页按钮与内容无溢出。
- 手动主题选择刷新后保持。
- 未保存选择时跟随系统主题变化。
- 打印预览为白底深色文字。

## 风险与回滚

- Ant Design 主题令牌可能改变部分组件默认间距或状态色，需要逐页回归。
- 轮播组件可能包含独立样式，应单独检查暗色状态。
- 主题状态迁移不涉及数据结构；如出现问题，可移除顶层 Provider 并恢复原条件类名。
