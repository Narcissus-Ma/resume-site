# 个人项目集卡片网格设计

## 背景

公开主页的“项目集”当前使用 Ant Design `Carousel`，一次只突出展示一个项目。项目数量增加后，用户需要逐个切换才能浏览全部内容，轮播还会复制节点用于循环播放，降低信息浏览效率。

本次改造参考用户提供的网站卡片布局，将项目集改为响应式网格卡片。视觉风格继续沿用当前站点的亮色、暗色主题、圆角、边框、阴影与蓝色强调色。

## 目标

- 移除项目集轮播，一次展示当前语言下的全部项目。
- 卡片采用上下两栏：上栏左侧项目图片、右侧项目标题；下栏展示项目描述。
- 整张有链接的卡片可点击，并在新标签页打开项目链接。
- 卡片提供清晰但克制的悬浮上移、边框高亮、阴影增强与图片缩放效果。
- 桌面、平板和移动端均保持整齐、可读且不产生横向滚动。
- 亮色、暗色、键盘操作和减少动画偏好均得到支持。

## 非目标

- 不修改 `Project` 数据结构或主页管理页面。
- 不新增项目分类、筛选、分页或加载更多功能。
- 不引入新的卡片库、动画库或网络资源。
- 不修改项目标题、描述、图片和链接数据。
- 不改造其他首页区块。

## 组件结构

保留 `ProjectsSection` 作为项目集区块组件，并在组件内部直接渲染纯展示卡片。此次卡片只在该区块使用，不提前拆分共享组件。

组件继续接收可选的 `projects` 属性，并保留现有默认项目数据作为未传值时的降级内容。项目数组为空时显示区块标题与副标题，但不渲染空卡片或轮播控件。

移除 `Carousel`、`Button` 和 `RightOutlined` 依赖。卡片内容使用语义化结构：

- 列表容器使用响应式 CSS Grid。
- 单个项目使用 `article` 表达独立内容。
- 有有效链接时，由外层 `a` 包裹完整卡片。
- 无有效链接时，仅渲染不可交互的 `article`，不伪装成按钮或链接。
- 项目图片使用项目标题作为替代文本。
- 标题使用层级不高于区块标题的语义化标题元素。

有效链接定义为去除首尾空白后非空、不等于 `#`，且解析后使用 HTTP 或 HTTPS 协议的字符串；站内相对地址按当前站点的 HTTPS 基准解析，因此仍可使用。`javascript:`、`data:` 等危险协议不生成锚点。有效链接使用 `target="_blank"` 和 `rel="noopener noreferrer"`，避免新标签页获得原页面控制权。

## 视觉设计

### 区块

项目集继续使用当前页面背景和居中标题结构。内容最大宽度从单个轮播所需的窄容器调整为适合多列卡片的宽容器，与技能卡片区域的信息密度保持一致。

### 卡片

每张卡片使用主题表面背景、主题边框和现有阴影变量：

- 圆角与站内其他卡片保持一致。
- 内边距在移动端略小，桌面端适度放大。
- 卡片高度由 Grid 拉齐，描述较短时仍保持整齐。
- 上栏使用 Flex，图片固定为紧凑矩形，标题占据剩余空间并允许自然换行。
- 图片使用 `object-fit: cover`，防止不同宽高比破坏布局。
- 下栏描述使用次级文字颜色和舒适行高。
- 标题和描述之间通过间距区分，不添加额外分割线。

### 交互状态

有链接的卡片：

- 默认显示指针光标。
- Hover 时向上移动约 4px，边框切换为主题强调色，阴影增强。
- Hover 时图片轻微放大，图片容器保持裁切。
- Focus-visible 时使用全局主题焦点环，并呈现与 Hover 相近的边框反馈。
- Active 时适当减小上移距离，提供按压反馈。

无链接的卡片：

- 保持正常卡片外观。
- Hover 时保留上浮、边框与图片缩放等纯视觉反馈。
- 不显示指针光标，避免误导用户认为可以点击。
- 不可通过 Tab 聚焦。

所有动画使用 CSS transition，并提供 `-webkit-` 前缀。全局已有 `prefers-reduced-motion` 规则会将过渡压缩到近乎即时，卡片不再添加循环动画。

## 响应式规则

- `< 640px`：单列卡片，上栏图片保持紧凑尺寸，标题可换行。
- `640px - 1023px`：两列卡片。
- `>= 1024px`：三列卡片。
- Grid 使用一致间距，卡片通过拉伸保持同一行等高。
- 容器和文字区域设置 `min-width: 0`，长标题与描述不得撑宽页面。
- 图片尺寸不随卡片无限放大，以保持参考截图中的“图标或缩略图 + 标题”视觉比例。

## 主题与样式组织

卡片专属样式放入 `projects-section.module.css`，避免继续扩大全局样式并防止影响其他 Ant Design 组件。样式只使用现有 CSS 变量，不硬编码亮暗主题颜色。

组件继续使用 Tailwind 处理区块级布局和标题间距；卡片内部结构、交互状态和浏览器前缀放入 CSS Module。这样既符合当前项目使用方式，也让复杂交互状态保持集中。

移除 `src/index.css` 对 `carousel.css` 的导入；如果确认没有其他组件使用轮播，则删除已失去用途的 `src/assets/styles/carousel.css`。

## 测试策略

新增项目卡片结构测试，覆盖：

- 组件源码不再导入或渲染 `Carousel`。
- 项目容器包含响应式一列、两列、三列网格规则。
- 有效项目链接使用新标签页和安全 `rel` 属性。
- `#` 或空链接不会生成可交互锚点。
- 卡片包含项目图片、标题和描述。

由于现有测试栈没有 React DOM 测试库，测试优先抽取一个小型纯函数判断项目链接是否有效，并用 Node.js test runner 验证边界；组件结构与样式通过 TypeScript、ESLint、构建和浏览器验收共同验证。不为此次改造额外引入测试依赖。

## 验收标准

### 自动检查

- 项目链接判断测试先失败后通过。
- `pnpm test`
- `pnpm typecheck`
- `pnpm lint`
- `pnpm format:check`
- `pnpm build`

### 浏览器检查

- 默认桌面视口下显示三列卡片，卡片上栏为图片加标题，下栏为描述。
- 约 768px 视口显示两列，卡片无重叠或横向溢出。
- 390px 视口显示单列，标题和描述正常换行。
- 亮色与暗色主题下边框、文字、阴影和焦点状态清晰。
- 鼠标悬浮时卡片上移且图片轻微缩放。
- 使用 Tab 可聚焦有链接卡片，按 Enter 在新标签页打开对应链接。
- 无有效链接的卡片不可点击、不可聚焦且无交互误导。
- 浏览器控制台无新增错误或警告。

## 风险与回滚

- 项目图片宽高比差异可能导致主体被裁切，需要通过固定容器和 `object-fit: cover` 平衡。
- 项目描述长度差异较大时，同一行卡片会按最长内容拉高，这是等高网格的预期行为。
- 删除轮播全局样式前需确认项目中没有其他 `Carousel` 使用；若仍有使用，则仅移除项目集组件依赖并保留全局文件。
- 改造不涉及数据迁移，回滚时恢复 `projects-section.tsx` 与轮播样式导入即可。
